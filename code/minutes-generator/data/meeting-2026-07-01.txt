\ Demo from Muhsin (about the use of DPV for recording AI output uses) -\ upcoming concepts in 2.4

Muhsin: we have a Chrome extension for Google Docs to record text that is generated by AI, the level of AI generation. It keeps track of what has been modified in the backend. This can be exported with a nice visualisation. The output is an annotated document that shows which text was AI generated, which was modified, and metadata about 'events'. There is also a log of whether the AI suggestions were accepted, were modified, etc. This is available right now. We are working on a beta next week. We are making a list on people who are interested and who have opinions.
Harsh: How do you use the concepts from DPV here? We had discussions on tracking the AI outputs, so want to see how DPV concepts look in the demo.
Muhsin: metadata is in JSON format, not fully aligned with DPV yet, but it will be with the release / final version
Sal: Can this be used for coding as well?
Muhsin: Haven't tested for coding yet, current focus is on text, but it could work for coding as well
Ruhi: what is the rate of error e.g. if we accuse a student of cheating or using AI, and we use other AI checking websites they have high error rates. What would it be if we used this for student assignment submissions?
Muhsin: This is recording while writing rather than evaluating a submission i.e. this is not detection. We are working on tamper-proofing it.
Harsh: Maybe a good idea to send an email to the mailing list explaining your work and what you want participants to do, and also how this is using the DPV concepts.
